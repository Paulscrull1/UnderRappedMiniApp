# UnderRapped / Web Music Bot — обзор проекта

---

## Что это за продукт

**Telegram-бот + Telegram Mini App** вокруг музыки: пользователи слушают треки (Яндекс.Музыка и SoundCloud), оценивают их по пяти критериям, копят опыт и уровень, соревнуются в лидерборде, ведут плейлист и профиль. Есть **игровая механика** (EXP, уровни, достижения) и **социальный слой** (общие топы, рецензии, шаринг треков в Telegram).

Продукт можно позиционировать как **«музыкальная игра с оценками»** или **«сообщество вкуса в музыке»** внутри экосистемы Telegram без отдельного приложения в сторах.

## Ценность для покупателя / инвестора

| Аспект | Суть |
|--------|------|
| **Готовый стек** | Рабочий бот, веб-Mini App, REST API, БД, интеграции с двумя музыкальными источниками |
| **Монетизация** | Заготовки под премиум, рекламу, партнёрки со стримингами; аудитория уже в Telegram |
| **Масштабирование** | SQLite → при росте можно заменить на PostgreSQL; API отделён от бота |
| **Уникальность** | Единый `track_id` (Яндекс + SoundCloud), встроенный плеер в Mini App, геймификация |

## Функционал (кратко)

### В Telegram-боте

- Старт, меню, поиск треков, чарт Яндекса, трек дня, плейлист по ссылке
- Карточка трека: оценка, избранное, скачивание (где доступно), переход в Mini App
- Мои оценки, плейлист, «мои скачанные» (при настройке storage-чата)
- Глобальная статистика и ленты рецензий
- Профиль: ник, статус, аватар (фото/эмодзи), закреплённый трек, лидерборд
- Команды (`/chart`, `/daily`, `/search`, приглашения и т.д.)
- Inline-поиск и шаринг треков по deep link (`start=track_...`)

### В Mini App (веб-интерфейс в Telegram)

- Главная: трек дня, топ по оценкам сообщества
- Чарт Яндекс.Музыки, поиск по Яндексу и SoundCloud
- Плейлист по URL Яндекса, раздел «Моё» (оценки + плейлист)
- Профиль: уровень, EXP, прогресс, достижения, лидерборд, редактирование профиля
- **Встроенный плеер**: стрим аудио через API, очередь из текущего списка, предыдущий/следующий трек, автопереход, громкость
- Оценка треков по критериям, рецензии, избранное, «поделиться» треком

### Ключевые модули

| Компонент | Назначение |
|-----------|------------|
| `main.py` | Точка входа бота: хендлеры команд и callback |
| `api_main.py` | FastAPI: эндпоинты `/api/*` для Mini App |
| `database.py` | SQLite: схема, миграции колонок, запросы |
| `music_providers.py` | Единая точка: трек по `id`, поиск (Яндекс + SoundCloud) |
| `yandex_music_service.py` | Чарт, дневной трек, плейлист, поиск, скачивание байтов |
| `soundcloud_service.py` | Поиск, метаданные, скачивание для плеера/бота |
| `telegram_validation.py` | Проверка `initData` Telegram для API |
| `handlers/*` | Логика сценариев бота (поиск, профиль, карточки и т.д.) |
| `miniapp_static/index.html` | Mini App: UI, плеер, вызовы API |

### Формат трека

- **Яндекс**: `track_id` в виде `id:album_id` (строка).
- **SoundCloud**: префикс `sc_` + идентификатор; роутинг в `music_providers.py` и в `/api/audio/{track_id}`.

## Как это работает (важные потоки)

1. **Пользователь открывает Mini App из бота** → Telegram передаёт `initData` → фронт шлёт его в заголовке `X-Telegram-Init-Data` → сервер валидирует подпись и узнаёт `user_id` → все персональные запросы привязаны к этому пользователю.

2. **Оценка трека** → POST `/api/reviews` с пятью шкалами (например: рифмы, ритм, стиль, харизма, вайб) → пересчёт суммы, начисление EXP, уровень, возможное открытие достижения → запись в `reviews` и `user_progress`.

3. **Прослушивание в Mini App** → GET `/api/audio/{track_id}` → для `sc_*` — SoundCloud, иначе Яндекс → поток `audio/mpeg` в `<audio>` на клиенте.

4. **Бот и Mini App** делят одну БД: оценки и плейлист видны и там, и там.

## Безопасность и конфигурация

- Секреты только из переменных окружения / `.env` (`TELEGRAM_BOT_TOKEN`, `YANDEX_MUSIC_TOKEN`, `MINI_APP_URL`, опционально SoundCloud, `BOT_USERNAME`, `STORAGE_CHAT_ID`).
- API для действий пользователя требует валидный `initData` (кроме публичных эндпоинтов вроде конфига).
